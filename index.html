How to Use This Website

There is no single right way to solve these. The point is to learn how to learn.

0
Use git and push to a remote

Version control is non-negotiable. Commit early and often, and push your work to a remote host like GitHub or GitLab. Every finished project becomes a portfolio piece.

1
Understand the problem

Research is the most transferable skill you can develop. When a problem feels overwhelming, break it down. Sketch it, write pseudocode, dissect it piece by piece. If it feels too easy, invent constraints, add features, or merge it with another idea.

2
Implementation

Start with a language you know. If that makes it too easy, switch to one you have never used. Write ugly code. Nobody cares. Get something working first, then refactor and polish. Done is better than perfect.

3
Leverage AI tools

Modern AI can accelerate your learning. Use it to plan your architecture before you write code, read through generated code to pick up new patterns, and debug errors faster. Always understand what the code does before you use it. AI is a lever, not a crutch.

4
Test

Think like an attacker. Write tests that try to break your own code. Find bugs, fix them, and document everything. Each cycle makes you sharper. Commit your tests alongside your implementation.

5
Re-Implement

Port your working solution to another language. Match the original behavior exactly, then go further. Experiment with new patterns, libraries, and architectural styles. Each rewrite deepens your understanding.

6
Don't stop learning

This is a marathon, not a sprint. Keep a journal, a blog, or a GitHub streak. Reflect on what you built, what went wrong, and what you would do differently. Then pick the next challenge and start again.

7
Resources

Wikipedia, Stack Overflow, and Google are all you will ever need. If you cannot find the answer, refine your search terms. If it truly does not exist, ask. And when someone helps you, write the answer down publicly so the next person finds it.

8
Classic texts
  • The Art of Computer Programming by Knuth
  • The Algorithm Design Manual by Skiena
  • Introduction to Algorithms by Cormen et al.
  • Artificial Intelligence: A Modern Approach by Russell
  • Structure and Interpretation of Computer Programs by Abelson
